# AI Agent 通识课教学内容清单

版本日期：2026-07-05  
课程定位：不是教你追 AI 热词，而是让你理解AI产品是怎么设计的，能力边界在哪里，教你真正把 AI agent 用到工作和生活里。

---

## 一、为什么需要这门课

很多人现在对 AI 很焦虑。

每个月都能听到新的技术名词：

```text
大模型、Agent、上下文、记忆、MCP、Skill、Token、 openclaw、claude、trae
```
没有技术背景，看不懂ai项目的源代码，只能听自媒体讲个模糊的概念，
别人说 AI 赋能让工作效率提高了几倍，自己却还停留在：

```text
打开一个聊天框
问几个问题
让 AI 写一段话
然后不知道还能做什么
```

这就像买了一台智能手机，却只拿来打电话。

这门课要解决的就是这个问题。

---

## 二、这门课到底学什么

一句话：

> 从“会和 AI 聊天”，升级到完全理解AI的能力边界，指挥 AI 完成电脑上能做的各种任务。

课程分成两条线：

| 线索 | 学什么 | 为什么重要 |
|---|---|---|
| 理论线 | AI agent的系统是怎么设计的，如何工作 | 理解ai底层原理，不再被技术名词唬住。学会如何充分利用ai的能力，未来出现任何新的ai产品都能快速上手 |
| 实践线 | 用 Codex 完成真实任务 | 加深对agent能力的实感，提升与ai协同工作的熟练度 |

本课会用 Codex 做主要练习工具，但目标不是让你只会用 Codex。

真正的目标是：

> 让你理解 AI agent 的通用工作方法与能力边界。任何一个 agent 产品你都能快速上手。

---

## 三、学完后你应该能做到什么

学完后，你应该能做到：

- 听到“大模型、agent、上下文、MCP、skill”这些词，不再一头雾水；
- 知道普通 chatbot 和 AI agent 的区别；
- 知道 AI 为什么会答错、什么时候不能信；
- 会把需求说清楚，让 AI 更好地帮你完成任务；
- 会让 Agent 帮你整理电脑文件，处理表格、分析数据、生成报告、管理邮件和待办；
- 会让 Agent 做一个简单网页或小应用原型；
- 怎么节省AI产品的使用开销，国产模型替代方案与KV Cache缓存机制的利用；
- 知道如何通过手机上的应用（Codex,微信,飞书等）远程控制Agent工作。

## 四、理论部分：把 AI 热词讲成人话

### 1. 大模型是什么

你会学到：

- 大模型不是神，也不是百科全书；
- 它的基本工作方式是：

```text
输入文字 -> 大模型 -> 输出文字
```

- 它回答问题时，本质上是在根据前面的内容，不断预测后面的内容；
- 它说得很流畅，不代表一定正确；
- 它常见犯错原因主要有三类：
  - 背景信息不全；
  - 问题没有边界；
  - 问题太复杂，推理链条太长。

你会明白：

> AI 不是“懂了一切”，而是“很会根据已有内容生成看起来合理的答案”。

### 2. AI Agent 是什么

普通 chatbot 更像：

```text
你问一句 -> 它答一句
```

AI agent 更像：

```text
你给一个目标 -> 它拆步骤 -> 它用工具 -> 它产出结果 -> 你检查反馈 -> 它继续修改
```

你会学到：

- Agent 不是更高级的聊天框；
- Agent 是一个能围绕目标做事的 AI 助理；
- 它可以读取资料、整理文件、分析表格、生成报告、修改项目、检查结果；
- 它不是自动正确，仍然需要人来判断和确认。

你会明白：

> 大模型负责“生成和推理”，agent 负责“带着工具去做事”。

### 3. 上下文是什么

很多人用 AI 不稳定，是因为没有理解“上下文”。

上下文可以先理解成：

> AI 当前能看见的所有信息。

包括：

- 你刚说的话；
- 前面的聊天记录；
- 你上传的文件；
- 你给的要求；
- 工具刚刚查到或算出的结果。

你会学到：

- AI 不是永远记得一切；
- 它当前看不见的信息，就只能猜；
- 重要要求要反复说清楚；
- 文件太多、任务太长时，要帮 AI 抓重点。

### 4. 记忆系统是什么

记忆系统可以理解成：

> AI 的长期小笔记本。

它可能记住：

- 你的语言偏好；
- 你的工作习惯；
- 你常用的格式；
- 某个长期项目的背景。

但你也会学到：

- 记忆不是万能；
- 记忆可能不完整；
- 重要事情不要只指望 AI “应该记得”；
- 关键任务仍然要重新说明要求。

### 5. MCP、CLI、工具扩展是什么

很多人觉得 AI 只是会聊天，是因为他用的 AI 没有接工具。

工具让 agent 能做更多事：

- 查网页；
- 读文件；
- 改文件；
- 算表格；
- 生成图表；
- 打开网页测试；
- 连接邮箱、日历、文档、知识库。

MCP 可以先理解成：

> 给 AI agent 接更多工具的一种方法。

CLI 可以先理解成：

> 用文字命令操作电脑的一种方式。

你不需要学会写复杂命令，但你要知道：

> Agent 能力的上限，很大程度取决于它能使用哪些工具。

### 6. Skill 是什么

Skill 可以理解成：

> 给 AI agent 看的“工作流程卡”。

比如做数据分析，不能一上来就写结论。更好的流程是：

```text
先看数据 -> 检查缺失 -> 统计 -> 画图 -> 总结 -> 检查结论
```

Skill 的作用就是把这种经验写下来，让 agent 做事更稳定。

你会学到：

- Skill 不是魔法；
- Skill 是标准流程；
- 重复任务适合做成 skill；
- 好 skill 能减少漏步骤、乱发挥和低质量输出。

### 7. 定时任务是什么

定时任务可以理解成：

> 让 AI 到时间提醒你、检查东西，或继续完成某个任务。

比如：

- 每天晚上提醒我复习；
- 每周五整理本周学习笔记；
- 每天检查一个网页是否更新；
- 到截止日期前提醒我提交材料。

你会学到：

- 自动化任务要有清楚目标；
- 要有停止条件；
- 高风险任务不要直接全自动；
- 先从提醒、总结、检查开始用。

### 8. AI 的能力边界

这部分非常重要。

你会学到：

- AI 会编；
- AI 会错；
- AI 会说得很像真的；
- AI 不替你负责。

以下场景不能完全相信 AI：

- 医疗；
- 法律；
- 金融；
- 合同；
- 学术引用；
- 隐私数据；
- 涉及钱和安全的决定。

核心原则：

> AI 可以帮你提高效率，但最终判断必须在人。

## 五、沟通部分：怎么让 AI 更好地帮你

很多人用 AI 效果差，不是因为 AI 不行，而是任务说得太模糊。

### 1. 好任务说明长什么样

以后你给 AI 任务时，要尽量说清楚：

```text
我要完成什么？
给谁看？
我有什么材料？
希望输出成什么格式？
有什么限制？
哪些地方不能乱编？
完成后怎么检查？
```

### 2. 三个最重要的沟通习惯

| 习惯 | 为什么重要 |
|---|---|
| 给背景 | 不给背景，AI 就会自己猜 |
| 说边界 | 不说边界，AI 就会给空泛答案 |
| 拆小步 | 任务太大，AI 容易中途跑偏 |

### 3. 常用句式

你会反复练习这些话：

```text
请先不要直接做，先列计划。
```

```text
请只根据我提供的资料回答。
```

```text
如果不确定，请明确说不确定。
```

```text
请把事实、猜测和建议分开。
```

```text
请一步一步来，每一步完成后先检查。
```

```text
请用普通人能听懂的话解释。
```

```text
请告诉我你还需要哪些信息。
```

## 六、实践部分：先把 Codex 这个 Agent 用明白

这门课不会只让你听概念。

我们会用 Codex 做练习，是因为 Codex 很适合让普通人看见 agent 怎么工作：

```text
它能看文件
能改文件
能运行工具
能检查结果
能根据反馈继续改
```

学 Codex 不是为了只会用 Codex，而是为了通过 Codex 理解一类 agent 产品的通用用法。

### 1. Codex 的原生工具能力

Codex 不是普通聊天框。它有一些“原生工具能力”，这些能力让它能在电脑里真正做事。

你会认识这些能力：

| 能力 | 普通人怎么理解 | 能做什么 |
|---|---|---|
| 读文件 | 打开资料看内容 | 看文档、表格、代码、说明书 |
| 写文件 | 帮你改或新建文件 | 写报告、整理表格、生成网页文件 |
| 搜文件 | 在一堆文件里找东西 | 找某个关键词、某类文件、某段内容 |
| Bash / 命令行 | 用文字命令操作电脑 | 批量处理文件、运行检查、启动小项目 |
| 浏览器 / 网页测试 | 打开网页看效果 | 检查页面能不能用、按钮能不能点 |

不同账号、版本和课程环境里，可用工具可能不完全一样。课堂重点不是背某个按钮，而是理解 agent 为什么能通过工具做事。

课堂不会要求学生背命令。重点是让学生知道：

> Codex 不只是“回答问题”，它可以通过工具读、写、查、改、运行和检查。

### 2. 由用户控制的 Codex 常用功能

课程会教你控制这些关键功能：

| 用户操作 | 人话解释 | 为什么重要 |
|---|---|---|
| 切换模型 | 选择更快、更便宜或更强的模型 | 不同任务不一定需要最贵模型 |
| 工具使用权限 | 决定 Codex 能不能读文件、改文件、运行命令 | 防止误操作和隐私风险 |
| 会话管理 | 新建、继续、归档、拆分任务 | 长任务不混乱，旧任务可追踪 |
| 任务计划确认 | 先让 Codex 列计划，再决定是否执行 | 避免它一上来就乱改 |
| 中途纠偏 | 发现方向不对，及时打断和改要求 | 让任务回到正确方向 |
| `/goal` 超大任务布置 | 把一个很大的目标交给 agent 持续推进 | 适合长任务，但更需要拆解和检查 |

你会练习的常用说法：

```text
请先不要动文件，先告诉我你准备怎么做。
```

```text
这个任务比较大，请先拆成几个阶段。
```

```text
只允许读取文件，暂时不要修改。
```

```text
改完后请告诉我你改了哪些文件、为什么这么改。
```

```text
请检查一遍有没有遗漏、重复、格式错误或明显不合理的地方。
```

### 3. 案例一：电脑文件整理

适合人群：

- 文件夹很乱；
- 截图、文档、表格、PDF 到处都是；
- 想整理但一直拖延。

你会学：

- 让 Codex 查看文件；
- 让它先提出分类方案；
- 人确认后再整理；
- 生成整理报告；
- 避免误删重要文件。

### 4. 案例二：数据分析处理

适合人群：

- 手里有 Excel 或 CSV；
- 想看趋势、找规律、写总结；
- 不会复杂数据分析工具。

你会学：

- 让 Codex 读取表格；
- 检查空值和异常值；
- 生成基础统计；
- 分析数据，编写研报；


### 5. 案例三：个人电子邮箱管理

适合人群：

- 邮件太多；
- 重要信息容易漏；
- 不知道哪些邮件需要回复；
- 想让 AI 帮忙整理待办。

课堂会优先使用模拟邮箱或脱敏邮件，不建议一开始直接接入真实私人邮箱。

你会学：

- 让 AI 给邮件分类；
- 提取待办事项；
- 总结重要邮件；
- 起草回复；

### 6. 案例四：小型 App 编写

很多人听到“写 app”会害怕，以为必须先学编程。

这门课不会让你背代码语法，而是训练你把需求说清楚，让 Codex 帮你做一个小原型。

适合做：

- 简单报名页；
- 学习打卡页；
- 个人作品展示页；
- 问卷结果看板；
- 小型计算器；
- 简单库存记录页。

不适合一开始做：

- 淘宝级 app；
- 微信级 app；
- 支付系统；
- 大型后台系统；
- 涉及真实用户隐私的系统。

你会学：

- 描述用户是谁；
- 描述页面上有什么；
- 描述按钮能做什么；
- 先做最小版本；
- 打开试用；
- 提出具体修改意见。

## 七、进阶部分：Agent 产品都会越来越像这些设计

这一章不是只讲 Codex，而是讲未来很多 agent 产品都会有的通用设计。

你现在听到这些词可能会觉得吓人：

```text
Skill、MCP、记忆系统、定时任务、手机控制
```

但它们背后的意思并不复杂。

### 1. Skill：把经验变成流程卡

Skill 可以理解成：

> 给 agent 看的“工作流程卡”。

比如数据分析不应该一上来就写结论，而应该：

```text
看数据 -> 查缺失 -> 做统计 -> 画图 -> 写结论 -> 检查风险
```

你会学到：

- 什么任务适合做成 skill；
- 一个好 skill 应该包含哪些步骤；
- skill 为什么能让 agent 少漏步骤；
- skill 为什么不是万能外挂。

课堂可能演示：

- 数据分析 skill；
- 文件整理 skill；
- 邮件整理 skill；
- 文章写作 skill；
- 课程资料整理 skill。

### 2. MCP：给 agent 接更多工具

MCP 可以理解成：

> 让 agent 连接外部工具和资料的一种方式。

没有工具时，agent 很多时候只能“猜”。

接上工具后，它可以：

- 读真实文件；
- 查真实数据；
- 连接表格；
- 连接日历；
- 连接知识库；
- 连接项目管理工具。


### 3. 记忆系统：让 agent 记住长期偏好

记忆系统可以理解成：

> agent 的长期小笔记本。

它可以帮助 agent 记住：

- 你喜欢什么格式；
- 你的常用工作流程；
- 某个项目的长期背景；
- 你反复强调过的偏好。

但课程也会强调：

- 记忆不是万能；
- 记忆可能不完整；
- 重要任务仍然要重新说明；
- 隐私信息不能随便让 AI 长期保存。

### 4. 定时任务：让 agent 到时间自动干活

定时任务可以理解成：

> 让 agent 到点提醒、检查或继续执行任务。

比如：

- 每天提醒复习；
- 每周整理一次学习笔记；
- 每天检查某个网页是否更新；
- 到截止日期前提醒提交材料；
- 定期生成工作摘要。


### 5. 手机控制：离开办公室依然能遥控 agent 工作

真实工作中，人不可能一直守在电脑前。

手机控制的价值是：

- 在路上查看任务进度；
- 用微信或飞书之类的通讯软件补充要求；
- 在手机上确认方向；
- 让电脑上的 Codex 继续处理文件；
- 把结果同步给团队。

## 八、省钱技巧

很多人用 AI 的另一个痛点是贵。

这部分会讲一些更接近“熟练用户”的省钱方法，但仍然用普通人能听懂的方式讲。

### 1. Agent系统+国产模型替代：不一定非要用最贵的 SOTA 模型

很多人一提 AI，就觉得一定要用 ChatGPT、Claude 这类最强模型。

但对普通用户来说，很多任务没有必要一上来就用最贵、最顶级的模型。

现在不少国产模型已经非常聪明。例如 GLM、DeepSeek、Kimi 等模型，在普通办公、数据处理、代码生成、数学推理、资料总结等场景里，已经能完成很多实际任务。

如果你不是程序员、数学研究员，或者不是在做特别高难度、特别高风险的任务，就不一定非要追求最强的 SOTA 模型。

可以先这样理解：

| 任务 | 是否一定要用最强模型 |
|---|---|
| 普通资料总结 | 不一定 |
| 文案初稿、标题、提纲 | 不一定 |
| Excel 数据处理和分析 | 不一定 |
| 普通网页、小工具、小应用开发 | 不一定 |
| 常见数学题、逻辑题 | 不一定 |
| 高难度科研、复杂软件工程、关键决策 | 更适合用强模型，并且要人工复核 |

课程会介绍一些替代方案，例如在课程环境里基于 cc switch、codex++ 等方式，让 Codex 类 agent 使用 GLM 5.2、Kimi K2.7、DeepSeek 等国产模型。具体版本以课程时实际可用版本为准。

这里要理解的不是“一个任务必须混用多个模型”，而是一个更简单的原则：

> 普通任务可以用足够聪明、价格更低的模型；只有特别复杂、特别关键的任务，才需要优先考虑最强模型。

这样做的意义是：

- 降低长期使用成本；
- 减少对单一海外模型的依赖；
- 让普通用户更容易把 AI 用成日常工具；
- 根据任务难度选择合适模型，而不是盲目追求最贵模型。

### 2. KV Cache：让重复内容不要反复花钱

KV Cache 听起来很技术，但可以先这样理解：

> AI 看过的前文，如果下次还能复用，就有机会少算一部分。

这有点像：

```text
第一次读一本资料很慢
第二次如果还能翻到书签，就不用从头再读
```

提高缓存命中率，就是尽量让 AI 更容易复用前面已经看过的内容。

普通人可以先掌握几个习惯：

- 固定常用系统提示词；
- 固定项目说明格式；
- 不要每次都大幅改开头；
- 把长期不变的背景放在前面；
- 把本次变化的要求放在后面；
- 同类任务尽量使用同一套模板。

你不需要成为工程师，也能理解这个原则：

> 任务格式越稳定，越容易复用；越是每次乱写，越难省钱。

## 九、这门课适合谁

适合：

- 听过很多 AI 热词，但一直没真正搞懂的人；
- 每天只会和 chatbot 聊天，但想让 AI 做实际工作的人；
- 没有编程基础，但想理解 agent 的人；
- 想提高办公效率，但不想学复杂技术的人；
- 对 AI 有焦虑感，想补上基础认知的人。
